# Формат имен файлов для проекта Geology Traps

## Структура именования

Формат: `{number}_{x|y}_{type}_{name}.png`

### Компоненты:
- **number**: Номер карты (например, `001`, `002`, `003`)
- **role**: Роль файла - `x` (входные данные) или `y` (целевая маска)
- **type**: Тип данных
- **name**: Название месторождения (например, `H150`, `BZ24`, `Field_A`)

### Типы файлов

#### Входные данные (x):
| type | Описание | Канал в модели |
|------|----------|----------------|
| `structuralNOisoline` | RGB карта без изолиний | rgb (3 канала) |
| `structuralBlackWhite` | Черно-белая карта глубины | depth_norm (1 канал) |
| `faults` | Карта разломов (опционально) | faults (1 канал) |

#### Целевые данные (y):
| type | Описание | Канал в модели |
|------|----------|----------------|
| `traps` | Карта ловушек (таргет) | traps (1 канал) |

## Примеры имен файлов

### Один семпл (месторождение H150, карта 001):
```
001_x_structuralNOisoline_H150.png      # RGB карта
001_x_structuralBlackWhite_H150.png     # Depth нормализованный
001_x_faults_H150.png                   # Разломы (опционально)
001_y_traps_H150.png                    # Ловушки (таргет)
```

### Другой номер карты того же месторождения - ДРУГОЙ семпл:
```
002_x_structuralNOisoline_H150.png
002_x_structuralBlackWhite_H150.png
002_x_faults_H150.png
002_y_traps_H150.png
```

### Другое месторождение - ДРУГОЙ семпл:
```
001_x_structuralNOisoline_H100.png
001_x_structuralBlackWhite_H100.png
001_x_faults_H100.png
001_y_traps_H100.png
```

## Группировка данных

### Семпл
Семпл определяется комбинацией `{number}_{name}`:
- `001_H150` = все 4 файла с number=001 и name=H150
- `002_H150` = отдельный семпл (другая карта того же месторождения)
- `001_H100` = отдельный семпл (другое месторождение)

### Разделение на train/val/test
Разделение происходит по **месторождениям (name)**, чтобы:
- Все карты одного месторождения попадали в одну выборку
- Модель не видела данные из тестовых месторождений при обучении

Пример:
- Если `H150` попал в test, то ВСЕ карты (`001_H150`, `002_H150`, ...) будут в test
- Если `H100` попал в train, то ВСЕ карты (`001_H100`, `002_H100`, ...) будут в train

## Конфигурация для варианта "без разломов"

Для обучения без учета разломов используются только 3 входных канала + target:
- `001_x_structuralNOisoline_H150.png` → rgb (3 канала)
- `001_x_structuralBlackWhite_H150.png` → depth_norm (1 канал)
- `001_y_traps_H150.png` → traps (target)

**Итого входных каналов**: 4 (RGB + depth)
**Маски**: map_mask (для игнорирования фона)

## Парсинг имен файлов

Функция `parse_filename()` в `data/dataloaders.py`:
```python
parse_filename("001_x_structuralNOisoline_H150.png")
# Returns: {
#     'number': '001',
#     'role': 'x',
#     'type': 'structuralNOisoline',
#     'name': 'H150'
# }
```

Функция `_parse_files()` в `dataset.py` группирует файлы по семплам.
